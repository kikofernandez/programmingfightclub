Programming Language Fight Club

In this editorial you can find in-progress and (at some point) finished books that compare two or more programming languages from a specific point of view: its type system.

All books are released under CC-SA-4.0: web format is always free; as soon as a book is finished, you can buy your copy on Amazon.com.

Feedback is always welcome. Thanks!

Ongoing books

GRASP principles for the Object-oriented mind

This books covers GRASP principles from the type system's perspective. We show the differences and aspects to consider when programming in a static and dynamic language. For these purposes, we show examples using Java and Python, comment on what the type system brings to an OOP perspective and show the benefits and drawbacks of these languages.

2017, Sep 12   —  Expected 2018, April

Lambda Calculus for the working programmer

This book introduces the lambda calculus (untyped and typed) and the programming language background necessary to understand the lambda calculus. Based on this gained knowledge, you will understand the formal syntax of almost any formalised language, such as web assembly.

Functional Programming: Haskell vs Clojure

Functional programming is gaining momentum but, which language should I choose? Scala, Erlang, Clojure, Haskell, Elm... all of them seem to be the same, somehow... In this book, we pick a static language such as Haskell and, as dynamic language we pick Clojure. We compare their advantages and disadvantages.
